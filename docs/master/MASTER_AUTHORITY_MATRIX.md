# BookTown Master Authority Matrix

This document is the canonical routing table for BookTown documentation. It identifies where runtime authority lives, where documentation authority currently lives, who owns each domain, which systems it depends on, what clients may do, what the backend owns, how AI may consume the domain, and what gaps remain.

This matrix does not create new architecture authority. It routes to current runtime and documentation authorities.

## Remaining Missing Authority Documents

No first-class system Master documents are currently missing. Lower-level architecture authorities remain needed for several domains, but Master-layer routing coverage is complete for the current system inventory.

## Authority Resolution Rules

1. Runtime authority is the implemented backend or contract source that owns writes, validation, and data truth.
2. Documentation authority is the highest current document that governs that runtime domain.
3. If a domain has both a register and an audit, the register governs and the audit provides evidence.
4. If a domain has a runbook and a registry entry, the registry governs status and the runbook governs operation.
5. If no documentation authority exists, this matrix records the gap instead of inventing authority.

## AI Routing Rule

AI systems must use this matrix before answering architecture, product, system, or governance questions. If a question maps to multiple domains, read the shared dependencies first and then the domain-specific authority documents.
